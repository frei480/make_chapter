# Make Chapter: Автоматическое создание оглавления видео из субтитров

Make Chapter — это инструмент на Python, который анализирует субтитры видео (файлы SRT) и генерирует структурированное оглавление с помощью OpenAI-совместимых языковых моделей. Проект предназначен для автоматизации разметки видео на разделы с таймкодами и ключевыми мыслями.

## Возможности

- **Чтение субтитров**: Поддерживает файлы в формате SRT.
- **Разбиение на чанки**: Интеллектуальное разделение длинных субтитров на части с учётом лимита токенов модели.
- **Суммаризация через LLM**: Использует OpenAI‑совместимый API (локальный или облачный) для создания оглавления.
- **Структурированный вывод**: Генерирует оглавление в формате Markdown с таймкодами и пунктами.
- **Асинхронная обработка**: Параллельная обработка нескольких файлов.

## Установка

### Предварительные требования

- Python 3.10 или выше
- [UV](https://github.com/astral-sh/uv) (рекомендуется) или pip
- Доступ к OpenAI‑совместимому API (например, [Ollama](https://ollama.com/), [OpenAI API](https://platform.openai.com/), [LM Studio](https://lmstudio.ai/) и т.п.)

### Клонирование репозитория

```bash
git clone <repository-url>
cd make_chapter
```

### Установка зависимостей

С помощью UV (уже настроен в проекте):

```bash
uv sync
```

Или с помощью pip (предварительно создайте `requirements.txt` из `pyproject.toml`):

```bash
uv pip compile pyproject.toml -o requirements.txt
pip install -r requirements.txt
```

### Настройка переменных окружения

Создайте файл `.env` в корне проекта и укажите ваш API‑ключ:

```env
OPENAI_API_KEY=your_api_key_here
```

Если используется локальный сервер (например, Ollama), ключ может быть произвольным, но должен присутствовать.

### Конфигурация

Основные настройки задаются в файле `settings.yaml`:

```yaml
main:
  OUTPUT_DIR: "d:/videoconv/output"        # Директория с исходными SRT‑файлами
  MODEL_NAME: "qwen/qwen3-vl-4b"           # Имя модели (должно поддерживаться вашим API)
  FILE_EXTENSION: "*.srt"                  # Расширение файлов субтитров
  OPENAI_BASE_URL: "http://127.0.0.1:1234/v1"  # Базовый URL API
  OPENAI_ENDPOINT: "http://127.0.0.1:1234/v1/chat/completions"
  TIKTOKEN_CACHE_DIR: "./cache"            # Директория для кэша токенизатора
  OPENAI_MAX_TOKENS: 12000                 # Максимальное количество токенов на запрос
```

Подстройте параметры под ваше окружение:
- `OUTPUT_DIR` — путь к папке, в которой лежат SRT‑файлы.
- `MODEL_NAME` — имя модели, которую будет использовать API.
- `OPENAI_BASE_URL` — адрес вашего OpenAI‑совместимого сервера.
- `OPENAI_MAX_TOKENS` — лимит токенов на один запрос (зависит от модели).
> **Важно**: Убедитесь, что путь `OUTPUT_DIR` существует и содержит SRT‑файлы. Если вы используете Windows, укажите путь в формате `C:/path/to/folder` или `D:/path`. Для Linux/macOS используйте стандартные пути, например `/home/user/videos`.


## Использование

1. Поместите SRT‑файлы в директорию, указанную в `OUTPUT_DIR`.
2. Запустите основной скрипт:

```bash
python main.py
```

Программа найдёт все файлы с расширением `.srt`, разобьёт каждый на чанки, отправит их в LLM и сохранит результат в файл с тем же именем, но расширением `.md`.

### Пример выходного файла

Для файла `video.srt` будет создан `video.md` с содержимым вида:

```
00:00 Введение в RAG
• объясняется тема видео
• вводится понятие Retrieval Augmented Generation
• объясняется зачем подключать базы знаний

05:30 Архитектура RAG
• описываются компоненты системы
• рассказывается про векторные базы данных
• объясняется процесс индексации и поиска
```
## Как это работает

1. **Сканирование директории**: Скрипт ищет все файлы с расширением `.srt` в указанной папке (`OUTPUT_DIR`).
2. **Чтение и токенизация**: Каждый файл читается, разбивается на логические блоки (по двойным переводам строк) и подсчитывается количество токенов с помощью кодировки `cl100k_base`.
3. **Разбиение на чанки**: Блоки группируются в чанки так, чтобы общее количество токенов в одном чанке не превышало `OPENAI_MAX_TOKENS`.
4. **Запрос к модели**: Для каждого чанка асинхронно отправляется запрос к OpenAI‑совместимому API с системным и пользовательским промптами.
5. **Сбор результатов**: Ответы модели (оглавления для каждого чанка) объединяются и сохраняются в Markdown‑файл с тем же именем, что и исходный SRT.

## Промпты

Промпты, используемые для взаимодействия с моделью, определены в `prompts.py`:

- **system_prompt** — задаёт роль и правила формирования оглавления.
- **user_prompt** — предоставляет пример и передаёт фрагмент субтитров.

При необходимости вы можете изменить промпты, чтобы адаптировать формат вывода под свои нужды.

## Настройка под разные API

### Ollama (локально)

1. Запустите Ollama и скачайте модель:

```bash
ollama pull qwen2.5:7b
```

2. В `settings.yaml` укажите:

```yaml
OPENAI_BASE_URL: "http://localhost:11434/v1"
MODEL_NAME: "qwen2.5:7b"
```

### OpenAI Cloud

```yaml
OPENAI_BASE_URL: "https://api.openai.com/v1"
MODEL_NAME: "gpt-4o-mini"
```

### LM Studio

```yaml
OPENAI_BASE_URL: "http://localhost:1234/v1"
MODEL_NAME: "local-model"  # Имя модели, указанное в LM Studio
```

## Ограничения

- Токенизация основана на кодировке `cl100k_base` (как у GPT‑4). Убедитесь, что ваша модель использует совместимую токенизацию.
- Длинные видео могут создавать много чанков, что увеличивает количество запросов к API.
- Качество оглавления зависит от способности модели понимать контекст и следовать инструкциям промпта.
